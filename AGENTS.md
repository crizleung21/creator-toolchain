# AGENTS.md

## Repository Identity

This repository is `creator-toolchain`, a Codex-native creator workflow system for planning, execution, state, rules, skill creation, evidence review, and plugin packaging.

## Operating Principles

- Use structured planning before non-trivial implementation.
- Treat `.agents/skills/` as the authoritative skill source.
- Generate `plugin/creator-toolchain/skills/` with `python3 scripts/sync_plugin_skills.py --write`; never hand-edit the mirror.
- Do not enable repository-local and installed plugin copies together.
- Use `creator-orchestrator` when workflow selection is unclear.
- Use `creator-intake-planner` for raw ideas and typed planning.
- Use `creator-execution-cycle` for implementation from an accepted plan.
- End every implementation cycle with Reconcile.
- Keep durable repository state inside the declared `.creator/*.json` surfaces.
- Do not silently mutate state unless the active workflow owns that surface.

## Package Contract

- Runtime package contents are defined by `scripts/package_integrity.py`.
- Package reports must match `docs/qa/package-integrity-report.json`.
- Release ZIP files must be generated by `scripts/build_plugin_package.py`.
- Do not package private state, caches, environment files, editor metadata, build archives, or operating-system artifacts.

## Validation

```bash
python3 -m unittest discover -s tests -p 'test_*.py' -v
python3 scripts/sync_plugin_skills.py --check
python3 scripts/package_integrity.py --root . --package-root plugin/creator-toolchain --check docs/qa/package-integrity-report.json
python3 scripts/validate_creator_toolchain.py --scope all
```

## Safety

- Hooks, MCP servers, app integrations, and plugin publishing are opt-in.
- Destructive file or remote operations require explicit user authorization.
- Never package user-private `.creator` state.
