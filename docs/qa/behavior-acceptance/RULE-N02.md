# Rule Preflight

## Primary workflow

`creator-rule-router` — `stage-proposal`, followed by explicit `approve-proposal`.

**Do-not-cross boundary:** A session observation cannot be promoted directly into a permanent rule without a clearly stated rule, scope, and approval. Project-specific fixture details must not become global policy.

## Matched Domains

| Domain | Reason | Rules Loaded |
|---|---|---:|
| Project execution | Fixture defines Phase 1 boundaries | 1 |
| Rule governance | Request seeks permanent promotion | 1 |

## Selected Rules

| Rule | Severity | Reason |
|---|---|---|
| Stage observations before permanence | Required | Prevents unreviewed policy changes |
| Require explicit approval | Required | Permanent rules affect future sessions |
| Keep Phase 1 registries out of scope | Fixture-scoped | Applies only to this character-image slide project |

## Non-Loaded Candidates

- Global creator rule: unsupported; the observation is project-specific.
- Persistent character-registry rule: conflicts with the fixture’s Phase 1 scope.

## Conflict Warnings

“Immediately promote” conflicts with the mandatory staging and approval gate.

## Next Action

Create a staged proposal such as:

> For this slide-deck project’s Phase 1, planning may document character-image assumptions, but must not implement persistent character or image registries.

The proposal remains non-permanent until explicitly approved. No fixture files are modified.
